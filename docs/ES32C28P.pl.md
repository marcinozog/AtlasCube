# ES3C28P / ES3N28P

*[English](ES32C28P.md) | Polski*

## Opis płytki

**ES3C28P** (wersja z dotykiem pojemnościowym) oraz **ES3N28P** (wersja bez dotyku) to zaawansowane moduły deweloperskie oparte na dwurdzeniowym mikrokontrolerze **ESP32-S3** z wbudowaną pamięcią **8 MB OPI PSRAM**. Centralnym punktem układu jest **2,8-calowy wyświetlacz IPS LCD** o rozdzielczości **240x320** pikseli, sterowany przez układ **ILI9341V**.

Dzięki bogatemu wyposażeniu płytka stanowi idealną bazę do projektów IoT, paneli HMI czy odtwarzaczy multimedialnych opartych na środowiskach natywnych (ESP-IDF/ESP-ADF).

**Główne cechy sprzętowe:**
* **Zasoby pamięci:** 16 MB zewnętrznej pamięci SPI Flash + 8 MB wbudowanego OPI PSRAM. Płytka posiada także gniazdo na **kartę MicroSD** połączoną przez szybki interfejs SDIO.
* **Rozbudowana sekcja Audio:** Płytka dysponuje zewnętrznym kodekiem I2S/I2C (ES8311), mikrofonem MEMS na dolnej stronie PCB oraz sprzętowym wzmacniaczem mocy (SC8002B), który obsługuje głośniki do 1.5W (8Ω) lub 2W (4Ω).
* **Zasilanie i bateria:** Gniazdo **USB Type-C** (symulujące port szeregowy oraz służące do zasilania). Dodatkowo wsparcie dla baterii Li-Po 3.7V ze sprzętowym kontrolerem ładowania oraz możliwością pomiaru napięcia przez wewnętrzny ADC mikrokontrolera.
* **Interfejsy:** Adresowalna dioda RGB, wyprowadzenia UART, magistrala I2C oraz złącze na 4 wolne piny GPIO do rozbudowy (Expansion IO).

## Układy scalone na płytce

* **Główny mikrokontroler:** `ESP32-S3` (zintegrowane 8M PSRAM).
* **Wyświetlacz:** Sterownik `ILI9341V` (SPI).
* **Dotyk (tylko ES3C28P):** Kontroler `FT6336G` (I2C, adres `0x38`).
* **Pamięć Flash:** `25VQ128` (16 MB SPI Flash).
* **Zarządzanie zasilaniem:** `TP4054` (Kontroler ładowania Li-Po do ~290mA).
* **Kodek Audio:** `ES8311` (I2C dla konfiguracji, I2S dla danych).
* **Wzmacniacz Audio:** `SC8002B`.

## Definicje pinów ESP32-S3

```c
// ==========================================
// WYŚWIETLACZ LCD (SPI) - ILI9341V
// ==========================================
#define LCD_PIN_CS    10  // LCD_CS
#define LCD_PIN_MOSI  11  // LCD_MOSI
#define LCD_PIN_CLK   12  // LCD_SCK (Zegar SPI)
#define LCD_PIN_MISO  13  // LCD_MISO
#define LCD_LED       45  // LCD_BL (Podświetlenie - High = Włączone)
#define LCD_PIN_DC    46  // LCD_RS (Data/Command - High = Data, Low = Command)
// Uwaga: Ekran i mikrokontroler współdzielą pin RESET (CHIP_PU). Należy to uwzględnić w konfiguracji (np. ustawić pin RST na -1 w kodzie).

// ==========================================
// KARTA SD (SDIO 4-bit)
// ==========================================
#define SD_CLK        38  // SDIO Clock
#define SD_CMD        40  // SDIO Command
#define SD_D0         39  // SDIO Data 0
#define SD_D1         41  // SDIO Data 1
#define SD_D2         48  // SDIO Data 2
#define SD_D3         47  // SDIO Data 3

// ==========================================
// AUDIO (I2S & Wzmacniacz)
// ==========================================
#define AUDIO_EN      1   // Wzmacniacz Audio (Low = Enable)
#define I2S_MCK       4   // Master Clock (MCLK)
#define I2S_SCK       5   // Bit Clock (BCLK)
#define I2S_DO        6   // Data Out (Do głośnika/DAC)   // ⚠ patrz "ERRATA"
#define I2S_LRC       7   // Word Select (L/R Clock)
#define I2S_DI        8   // Data In (Z mikrofonu)        // ⚠ patrz "ERRATA"

// ⚠ ERRATA — kolejność DO/DI w tej tabeli (z oryginalnej dokumentacji
// producenta) jest BŁĘDNA dla faktycznego wykonania PCB. Weryfikacja
// (oscyloskop + porównanie z demem "music.ino" Waveshare, które gra
// poprawnie) pokazuje:
//     ESP32 GPIO8  → ES8311 pin 9 (DSDIN)   = DAC playback IN
//     ESP32 GPIO6  ← ES8311 pin 7 (ASDOUT)  = ADC capture OUT
// W kodzie firmware'u używamy faktycznego mapowania:
//     I2S_DATA = 8   // dane do głośnika
//     I2S_DI   = 6   // dane z mikrofonu
// Patrz: main/include/defines.h, sekcja BOARD_ES3C28P.

// ==========================================
// DOTYK (FT6336G) & MAGISTRALA I2C
// ==========================================
// Magistrala I2C współdzielona dla Dotyku, Kodeka Audio i złącza rozszerzeń
#define I2C_SCL       15  // I2C Clock
#define I2C_SDA       16  // I2C Data
#define CTP_INT       17  // Przerwanie dotyku (Low level trigger)
#define CTP_RST       18  // Reset dotyku (Low level trigger)

// ==========================================
// PORT SZEREGOWY (Programowanie/UART0)
// ==========================================
#define UART_TXD0     43  // TXD0
#define UART_RXD0     44  // RXD0

// ==========================================
// POZOSTAŁE WEJŚCIA/WYJŚCIA (GPIO)
// ==========================================
#define BOOT_BTN      0   // Przycisk BOOT (Zejście w stan niski)
#define PIN_IO2       2   // Złącze rozszerzeń
#define PIN_IO3       3   // Złącze rozszerzeń
#define PIN_IO14      14  // Złącze rozszerzeń
#define PIN_IO21      21  // Złącze rozszerzeń
#define BAT_ADC       9   // Pomiar napięcia baterii (ADC)
#define RGB_LED       42  // Dioda trójkolorowa RGB (WS2812B / Neopixel)

// ==========================================
// USB (Natywne)
// ==========================================
#define USB_D_MINUS   19  // USB-
#define USB_D_PLUS    20  // USB+
```

## Ogólne wytyczne do integracji (ESP-IDF / ESP-ADF)

Aby poprawnie zainicjalizować peryferia tej płytki w natywnym środowisku Espressif, asystent AI/programista powinien postępować według poniższych wytycznych:

### 1. Konfiguracja sprzętowa pamięci (`sdkconfig`)
Ponieważ układ wspiera szybką pamięć w trybie **Octal SPI (OPI)**, wymagane jest uaktywnienie obsługi 8 MB PSRAM OPI. Należy włączyć w `menuconfig` / `sdkconfig`:
* **Flash:** 16 MB.
* **PSRAM:** Włączone wsparcie dla modułów wbudowanych (OPI), taktowanie odpowiednie dla ESP32-S3 (np. 80MHz lub 120MHz w zależności od stabilności). Piny dla PSRAM są zarezerwowane wewnętrznie (26, 27, 28, 30, 31, 32, 33, 34, 35, 36, 37) i nie wolno ich używać w logice aplikacji.

### 2. Wyświetlacz (ILI9341V) i Dotyk (FT6336G)
Korzystając z API `esp_lcd`:
* **LCD:** Panel podłączony jest przez interfejs 4-Wire SPI (nie korzysta z I2C ani równoległego!). Driver to `ILI9341`. Należy zainicjalizować sterownik `esp_lcd_panel_io_spi`, a następnie przekazać go do biblioteki docelowej (np. LVGL). Rozdzielczość to **240x320**.
* **Podświetlenie:** Pin GPIO45. Należy sterować nim za pomocą sygnału wysokiego (lub z użyciem LEDC PWM do regulacji jasności).
* **Dotyk (CTP):** Użyto kontrolera **FT6336G** współpracującego po magistrali I2C pod adresem **0x38**. Należy obsłużyć go dedykowanym sterownikiem kompatybilnym z serią FT6X36.

### 3. Pomiar napięcia baterii
Odczyt prowadzony jest przez wbudowany kontroler ADC na pinie **GPIO9**. Ponieważ sprzętowy dzielnik napięcia (rezystory) zmniejsza napięcie baterii o połowę (skala 1:1), odczytany przez framework wynik w miliwoltach **musi zostać pomnożony przez 2**, by uzyskać rzeczywiste napięcie akumulatora Li-Po.

### 4. Audio (ESP-ADF / I2S)
* **Wzmacniacz:** Uruchomienie głośnika wymaga ustawienia pinu **GPIO1 w stan niski (LOW)**, co uruchamia układ SC8002B.
* **Kodek ES8311:** Konfiguracja odbywa się przez wspólną z ekranem dotykowym szynę I2C (SDA: 16, SCL: 15). Transmisja dźwięku idzie natywnym interfejsem I2S.
* **Karta SD (SDIO):** Dla zapewnienia najwyższej wydajności, szczególnie przy odczycie multimediów czy zasobów graficznych, kartę SD należy zainicjalizować z wykorzystaniem protokołu **SDIO 4-bit** (nie zwykłego SPI z jednym przewodem danych).
