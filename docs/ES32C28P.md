# ES3C28P / ES3N28P

*English | [Polski](ES32C28P.pl.md)*

## Board overview

**ES3C28P** (capacitive-touch variant) and **ES3N28P** (no-touch variant) are
off-the-shelf development modules built around the dual-core **ESP32-S3**
microcontroller with **8 MB of in-package OPI PSRAM**. The centrepiece of the
module is a **2.8-inch IPS LCD** at **240×320** pixels, driven by an
**ILI9341V** controller.

Thanks to its on-board peripherals the board is a solid base for IoT projects,
HMI panels and small media players running on native Espressif tooling
(ESP-IDF / ESP-ADF).

**Main hardware features:**
* **Memory:** 16 MB external SPI flash + 8 MB on-package OPI PSRAM. A
  **MicroSD** slot wired for the high-throughput SDIO interface is also
  available.
* **Rich audio section:** External I2S/I2C codec (ES8311), MEMS microphone on
  the underside of the PCB, and a hardware power amplifier (SC8002B) that
  drives speakers up to 1.5 W (8 Ω) or 2 W (4 Ω).
* **Power and battery:** **USB Type-C** connector (acts as both serial port
  and power input). Built-in support for 3.7 V Li-Po batteries with a
  hardware charging controller; battery voltage is measurable through the
  ESP32-S3's internal ADC.
* **Interfaces:** Addressable RGB LED, UART pins, an I2C bus, and an
  expansion header with 4 spare GPIOs.

## On-board chips

* **Main MCU:** `ESP32-S3` (with 8 MB in-package PSRAM).
* **Display:** `ILI9341V` controller (SPI).
* **Touch (ES3C28P only):** `FT6336G` controller (I2C, address `0x38`).
* **Flash:** `25VQ128` (16 MB SPI flash).
* **Power management:** `TP4054` (Li-Po charging controller, ~290 mA).
* **Audio codec:** `ES8311` (I2C for configuration, I2S for audio data).
* **Audio amplifier:** `SC8002B`.

## ESP32-S3 pin map

```c
// ==========================================
// LCD (SPI) - ILI9341V
// ==========================================
#define LCD_PIN_CS    10  // LCD_CS
#define LCD_PIN_MOSI  11  // LCD_MOSI
#define LCD_PIN_CLK   12  // LCD_SCK (SPI clock)
#define LCD_PIN_MISO  13  // LCD_MISO
#define LCD_LED       45  // LCD_BL (backlight - High = ON)
#define LCD_PIN_DC    46  // LCD_RS (Data/Command - High = Data, Low = Command)
// Note: the LCD shares its RESET line with the MCU (CHIP_PU). Account for
// this in software (e.g. pass RST = -1 to the panel driver).

// ==========================================
// MicroSD (SDIO 4-bit)
// ==========================================
#define SD_CLK        38  // SDIO Clock
#define SD_CMD        40  // SDIO Command
#define SD_D0         39  // SDIO Data 0
#define SD_D1         41  // SDIO Data 1
#define SD_D2         48  // SDIO Data 2
#define SD_D3         47  // SDIO Data 3

// ==========================================
// AUDIO (I2S & amplifier)
// ==========================================
#define AUDIO_EN      1   // Audio amplifier (Low = Enable)
#define I2S_MCK       4   // Master Clock (MCLK)
#define I2S_SCK       5   // Bit Clock (BCLK)
#define I2S_DO        6   // Data Out (to speaker/DAC)   // ⚠ see "ERRATA"
#define I2S_LRC       7   // Word Select (L/R Clock)
#define I2S_DI        8   // Data In (from microphone)   // ⚠ see "ERRATA"

// ⚠ ERRATA — the DO/DI ordering in this table (taken from the original
// manufacturer documentation) is WRONG for the actual PCB. Verification
// (scope + comparison against the Waveshare "music.ino" demo, which plays
// correctly) shows:
//     ESP32 GPIO8  → ES8311 pin 9 (DSDIN)   = DAC playback IN
//     ESP32 GPIO6  ← ES8311 pin 7 (ASDOUT)  = ADC capture OUT
// The firmware therefore uses the real wiring:
//     I2S_DATA = 8   // speaker data
//     I2S_DI   = 6   // microphone data
// See: main/include/defines.h, BOARD_ES3C28P section.

// ==========================================
// TOUCH (FT6336G) & I2C BUS
// ==========================================
// Shared I2C bus — touch controller, audio codec and expansion header
#define I2C_SCL       15  // I2C Clock
#define I2C_SDA       16  // I2C Data
#define CTP_INT       17  // Touch interrupt (active low)
#define CTP_RST       18  // Touch reset (active low)

// ==========================================
// SERIAL PORT (programming / UART0)
// ==========================================
#define UART_TXD0     43  // TXD0
#define UART_RXD0     44  // RXD0

// ==========================================
// OTHER I/Os (GPIO)
// ==========================================
#define BOOT_BTN      0   // BOOT button (active low)
#define PIN_IO2       2   // Expansion header
#define PIN_IO3       3   // Expansion header
#define PIN_IO14     14   // Expansion header
#define PIN_IO21     21   // Expansion header
#define BAT_ADC       9   // Battery voltage measurement (ADC)
#define RGB_LED      42   // Addressable RGB LED (WS2812B / NeoPixel)

// ==========================================
// USB (native)
// ==========================================
#define USB_D_MINUS  19   // USB-
#define USB_D_PLUS   20   // USB+
```

## Integration notes (ESP-IDF / ESP-ADF)

To bring up this board's peripherals from native Espressif tooling, follow
the guidelines below.

### 1. Memory configuration (`sdkconfig`)
Because the ESP32-S3 on this module supports fast memory in **Octal SPI (OPI)**
mode, the 8 MB OPI PSRAM has to be enabled explicitly in `menuconfig` /
`sdkconfig`:
* **Flash:** 16 MB.
* **PSRAM:** Enable in-package PSRAM with the OPI driver, clocked to suit the
  ESP32-S3 (e.g. 80 MHz or 120 MHz depending on stability). The PSRAM GPIOs
  are reserved internally (26, 27, 28, 30, 31, 32, 33, 34, 35, 36, 37) and
  must not be reused by the application.

### 2. Display (ILI9341V) and touch (FT6336G)
Using the `esp_lcd` API:
* **LCD:** the panel is wired as 4-wire SPI (not I2C, not parallel). The
  driver is `ILI9341`. Initialize the panel via `esp_lcd_panel_io_spi`, then
  hand the panel handle to the target library (e.g. LVGL). Resolution is
  **240×320**.
* **Backlight:** GPIO45. Drive it high (or via LEDC PWM for brightness
  control).
* **Touch (CTP):** the **FT6336G** controller is on I2C at address **0x38**.
  Use a driver compatible with the FT6X36 family.

### 3. Battery voltage measurement
The battery is sampled by the internal ADC on **GPIO9**. The hardware divider
on the PCB halves the battery voltage (1:1 ratio), so the millivolt reading
returned by the framework **must be multiplied by 2** to recover the actual
Li-Po cell voltage.

### 4. Audio (ESP-ADF / I2S)
* **Amplifier:** to unmute the speaker, drive **GPIO1 LOW**. That enables
  the SC8002B power amplifier.
* **ES8311 codec:** configured via the I2C bus shared with the touch panel
  (SDA = 16, SCL = 15). Audio samples are streamed over the native I2S
  peripheral.
* **MicroSD (SDIO):** for maximum throughput, especially when reading media
  or graphics assets off the card, mount the card with the **SDIO 4-bit**
  protocol (not single-line SPI).
